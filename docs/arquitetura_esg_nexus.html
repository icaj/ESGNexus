Diagrama Geral de Arquitetura

Módulos de Software

🖥️ Frontend — Streamlit :8501

  • Interface web SPA em Python puro
  • cliente_api.py — cliente HTTP com Bearer JWT
  • Upload de planilha (.xlsx/.csv) com validação de colunas obrigatórias
  • Dashboard executivo (última avaliação por fornecedor)
  • Tela de histórico temporal por fornecedor
  • Visualização do plano de ação por avaliação

⚙️ Backend — FastAPI :8000

  • Uvicorn ASGI server, CORS liberado para Streamlit
  • Autenticação JWT HS256, expiração 480 min
  • Rotas: rotas_enterprise.py + rotas_publica.py
  • Schemas Pydantic v2 em esquemas.py
  • Persistência com sessao.flush() para FK chaining
  • Dashboard usa subquery MAX(id) GROUP BY fornecedor_id

🧠 esg_ml — Arquitetura Hexagonal

  • Domínio: Empresa, ScoreESG, DiagnosticoESG, ItemPlanoAcao
  • Aplicação: ClassificarFornecedor (use case principal)
  • Adaptadores: leitor_fornecedores_pandas.py — valida obrigatórias, silencia opcionais ausentes
  • Infraestrutura: SQLAlchemy ORM, JWT helper, logging estruturado
  • Pipeline CRISP-DM fases 2–6 encapsulado

🗄️ NeonDB — PostgreSQL 16

  • Banco compartilhado: API + MLflow + Airflow
  • Tabelas: usuarios, fornecedores, avaliacoes_esg, planos_acao (nova), experimentos_ml
  • Relacionamentos ORM: Fornecedor → Avaliações → Planos de Ação
  • Soft-delete via ondelete='SET NULL' nos FKs opcionais
  • Cascade delete: AvaliacaoBanco → PlanoAcaoBanco

📊 MLflow :5000

  • Experiment tracking com flag usar_mlflow=True
  • Armazenamento de artefatos .joblib (KNN + RF)
  • Backend store: PostgreSQL compartilhado (tabela mlflow_*)

✈️ Apache Airflow 2.9.3 :8080

  • DAG dag_treinamento_esg — schedule diário às 02:00
  • Task: cURL POST /treinar com credenciais de serviço
  • Metadados no PostgreSQL compartilhado

Modelo de Banco de Dados

Diagrama de Relacionamentos

fornecedores

Cadastro master do fornecedor. Upsert por (name, industry). Campos opcionais preservados no update se novo valor for nulo.

  • Relacionamento 1→N com avaliacoes_esg
  • Relacionamento 1→N com planos_acao (atalho)
  • atualizado_em via onupdate

avaliacoes_esg

Append-only: cada classificação cria um novo registro. Permite rastreio temporal da evolução ESG do fornecedor.

  • fornecedor_id nullable (retrocompat.)
  • Campos desnormalizados (name, industry) para consultas autônomas
  • Cascade delete → planos_acao

planos_acao novo

Um item por ação recomendada por pilar. Criado junto com cada avaliação via flush().

  • avaliacao_id obrigatório (CASCADE)
  • fornecedor_id opcional (SET NULL)
  • Ordenado por importancia DESC

Fluxos Principais

📤 Upload de Planilha + Classificação em Lote

1
Usuário faz upload da planilha via Streamlit → POST /avaliar/upload com Bearer JWT
2
leitor_fornecedores_pandas normaliza colunas, valida obrigatórias, coleta opcionais
3
Para cada linha: ClassificarFornecedor use case executa KNN + RF, gera DiagnosticoESG com plano_acao
4
_persistir_fornecedor() → upsert em fornecedores → sessao.flush() → obtém id
5
_persistir_avaliacao() → insert em avaliacoes_esg → sessao.flush() → obtém id
6
_persistir_plano_acao() → insert N itens em planos_acao com FKs corretas
7
sessao.commit() → resposta JSON com resultados + planos de ação

📋 Histórico e Plano de Ação por Fornecedor

1
Usuário seleciona fornecedor no Streamlit → GET /fornecedores/{id}/historico
2
API retorna HistoricoFornecedorSaida: total_avaliacoes + lista de avaliações ordenadas por data DESC
3
Cada HistoricoItemSaida inclui scores, grade, quadrante e plano_acao: list[PlanoAcaoSaida]
4
Para avaliação específica: GET /avaliacoes/{id}/plano-acao retorna itens ordenados por importancia DESC
5
Para plano atual: GET /fornecedores/{id}/plano-acao retorna plano da avaliação mais recente

🤖 Retreinamento Automático (Airflow)

1
Airflow scheduler dispara dag_treinamento_esg às 02:00 UTC diariamente
2
Task executa: curl -X POST http://api:8000/treinar -H "Authorization: Bearer {token}"
3
API inicia pipeline CRISP-DM: KaggleHub → pré-proc → GridSearchCV → avaliação acc≥93% / F1-Med≥88%
4
Artefatos .joblib salvos; experimento registrado em MLflow e experimentos_ml

🔐 Autenticação JWT

1
Streamlit envia POST /auth/token com credenciais → recebe JWT HS256
2
Token armazenado em st.session_state, incluído em todos os requests como Authorization: Bearer <token>
3
FastAPI Depends(get_current_user) valida token em cada endpoint protegido
4
Expiração em 480 min; renovação automática pelo cliente

Stack Tecnológico

Backend

  • Python 3.11+
  • FastAPI + Uvicorn
  • SQLAlchemy 2.x ORM
  • Pydantic v2
  • python-jose (JWT)
  • passlib (bcrypt)

ML / Data

  • scikit-learn (KNN, RF, GridSearchCV)
  • pandas (leitura de planilhas)
  • joblib (serialização de modelos)
  • kagglehub (dados de treino)
  • MLflow 2.x (tracking)

Infra / Frontend

  • Streamlit (frontend)
  • PostgreSQL 16 / NeonDB
  • Apache Airflow 2.9.3
  • Docker / docker-compose
  • uvicorn workers
